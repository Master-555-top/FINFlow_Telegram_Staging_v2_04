# FINFlow Design System v2.08

## Direction
Premium dark/cosmic finance cockpit. Keep FINFlow dark purple identity, but make it feel like a mature mobile app ecosystem.

## Reference principles
- Category first, detail second.
- Big rounded cards instead of dense technical feeds.
- Clear command headers per screen.
- Circular progress and dashboard visuals where they explain state.
- Bottom navigation must feel stable and touch-friendly.
- Do not expose private technical/security details in daily surfaces.

## Visual language
- Background: black/deep purple aurora.
- Accents: violet, magenta, cyan, occasional lime for positive status.
- Surfaces: glass panels, subtle borders, glow shadows.
- Typography: large confident numbers, short labels, less paragraph noise.

## Locked UX rule
FINFlow is a personal operating system. Every screen must answer: what should I do now, what is the current state, what is risky, and where do I go next?
