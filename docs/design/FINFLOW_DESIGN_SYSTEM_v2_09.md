# FINFlow Design System v2.09 — QA-ready ecosystem UX

## Purpose
FINFlow is being shaped as a personal operating system, not a set of loose panels. The System tab now has a dedicated QA route so every deploy can be verified in a predictable order.

## UX rules locked in v2.09
- Top-level System must stay category-first.
- QA must be visible as its own section, not hidden in Deploy.
- Every deploy should be checked through System → QA → What to check.
- Telegram/device checks stay separate from cloud/write checks.
- Cloud writes remain disabled until backup, RLS/security review and guarded conflict wizard are complete.

## Visual language
- Deep cosmic dark background.
- Glass panels with clear spacing and soft borders.
- Purple/magenta/cyan accents, not random colors.
- Large primary numbers and short labels.
- Mobile-first touch targets.
