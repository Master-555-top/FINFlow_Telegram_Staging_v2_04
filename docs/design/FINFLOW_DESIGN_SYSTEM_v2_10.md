# FINFlow Design System v2.10 — Compact Premium Mobile Cockpit

## Inputs
- User-provided Pinterest-style dark purple finance/gaming/security/mobile references.
- Apple-like hierarchy: fewer choices at first level, clear navigation, progressive disclosure.
- Material-style minimum touch targets and card hierarchy.

## Locked visual direction
- Dark cosmic base: black, deep purple, violet.
- Accents: neon violet, cyan, magenta, selective lime only for status.
- Glass cards with soft borders and glow, but not every element should glow equally.
- Mobile-first. Telegram Mini App viewport is the primary target.

## v2.10 rules
1. No long API strings as card headers on mobile. Use short labels and optional detail disclosure.
2. First level must be dashboard/cockpit, not documentation.
3. Explanatory text is allowed only behind details, checklist windows or helper states.
4. Every interactive target should feel finger-sized.
5. Runtime status cards must fit inside Telegram without horizontal scroll.
6. Every future screen pass must reduce visible text and improve hierarchy.
