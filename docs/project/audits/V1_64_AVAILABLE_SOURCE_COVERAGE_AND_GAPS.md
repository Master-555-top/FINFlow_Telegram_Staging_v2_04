# v1.64 — Available Source Coverage and Gaps

## What is available inside current package

The current package contains:
- changelog history from v1.1 to v1.64;
- transcript ledger;
- turn-by-turn operation ledger from v1.62 onward;
- project memory;
- current state;
- live reality register;
- requirements ledger;
- source recheck register;
- implementation notes;
- regression/build/security reports;
- source reviews;
- bank candidate CSV;
- typed app code.

## What is not perfectly available

Older exact assistant responses are not all fully copied verbatim into the package.

Older user messages before the strict ledger are partially available through:
- current chat source intake;
- transcript ledger;
- chronicle;
- project memory;
- summarized context files.

## Important honesty

From v1.62 onward the tracking is stricter.

Before v1.62, much of the history exists as:
- structured summaries;
- requirement ledgers;
- changelog entries;
- implementation files;
- source reviews;
- package reports.

That means a true full reconstruction from the absolute beginning must be described as:

```text
full available project-file reconstruction
```

not:

```text
perfect verbatim reconstruction of every historical assistant response
```

unless those exact messages are present in the package source files.

## Current corrective action

v1.64 adds an origin-to-current audit from all available version files and explicitly records this limitation.

Future packages must continue the v1.62 strict ledger.
