# FINFlow v3 — Day Core Input Model v1.24

## Назначение

Day Core Input Model — единый входной слой для расчёта дня.

FinFlow не должен считать день из разрозненных кусков UI. Все данные дня должны сначала приводиться к одной модели ввода, а уже потом использоваться для прогноза, AI-решения, фондов, целей и отчётов.

## Главная формула мышления

```
текущее время
+ деньги сейчас
+ смена / заказы / грязный доход
+ Drivee / топливо / рабочие расходы
+ обязательства
+ фонды
+ задачи дня
+ сон / еда / встречи / машина
= решение, что делать сегодня
```

## Статусы ввода

- `draft` — черновик.
- `review_needed` — есть значения, требующие подтверждения.
- `confirmed` — можно использовать в расчётах.
- `rejected` — не применять.
- `archived` — оставить в истории.

## Источники ввода

- manual;
- import_review_queue;
- bank_statement;
- taxi_log;
- ai_suggestion;
- system_default.

## Что входит в модель дня

### Деньги сейчас

- наличные;
- карта;
- баланс Drivee;
- зарезервировано / не свободно;
- заметка.

### Такси

- заказов сделано;
- грязными сделано;
- активные часы;
- полная смена;
- прогноз грязными к вечеру;
- цель грязными;
- цель чистыми;
- ставка Drivee;
- топливо;
- план километража.

### Обязательства

- машина 45 000 ₽ к 6 числу;
- банкротство/банк 15 000 ₽ к 15 числу;
- другие будущие обязательства.

### Фонды

- рабочий фонд;
- встречи;
- ремонт машины;
- подушка;
- переезд;
- временные цели.

### Задачи дня

- работа;
- еда;
- сон;
- встреча;
- машина;
- административные дела;
- проект;
- отдых.

## Критичное правило

Day Core не принимает банковскую операцию, строку из старого чата или старую запись как факт без статуса review/confirmed.
