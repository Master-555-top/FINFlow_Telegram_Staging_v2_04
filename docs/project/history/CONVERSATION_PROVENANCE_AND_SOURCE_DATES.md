Version: v1.12  
Updated: 2026-06-15 04:27

## Purpose

This file prevents confusion between:

1. The current active ChatGPT conversation.
2. Previous conversations referenced through memory.
3. Uploaded/exported text files from earlier chats.
4. Historical financial data inside Telegram/bank exports.

## Important correction

The **03.06.2026** FinFlow/Taxi Income Bot content is not the start date of the current active conversation.

It belongs to earlier project context / previous dialogue / uploaded historical materials about Taxi Income Bot and FinFlow analytics.

The current active working conversation, where the user uploaded many files, requested project memory archives, rebuild packages, and FinFlow v3 working packages, is treated as starting in the visible active context on **15.06.2026**.

## Source categories

### A. Current active conversation — 15.06.2026

Main events:
- user uploaded FinFlow files and historical data;
- full project reconstruction began;
- assistant created documents and working packages;
- archive versions v0.1 through v1.12 were produced;
- new requirements were added: real current balance, gross target, Toyota Premio, Drivee, car maintenance, calendar, 24-hour realism, live shift tracking, allocation recommendations, meetings 5/week, context-aware answers.

### B. Previous project conversations and imported context — before 15.06.2026

Examples:
- 03.06.2026 Taxi Income Bot / audit context;
- older bot code discussions;
- previous Telegram-style accounting;
- project notes and specs uploaded as files;
- old expense/income/shift histories;
- earlier AI protocol prompts.

These are valid project sources, but they are not the same as the current active chat timeline.

### C. Historical financial records

Examples:
- Telegram expense history;
- work/shift logs;
- bank PDF;
- pasted text files with March/April/May/June operations.

These should be normalized through Import Center / Review Queue before becoming production records.

## Timeline rule

Every future timeline file must label events as:

- Current chat event;
- Previous conversation / imported context;
- Historical financial/source data;
- Generated project artifact.

Do not merge them into one timeline without provenance labels.

## Why this matters

Incorrect provenance can cause confusion, for example:

- thinking that the current chat began on 03.06.2026;
- treating a previous bot-audit context as a current-chat message;
- mixing historical transaction dates with project decision dates.

FinFlow documentation must distinguish these clearly.
