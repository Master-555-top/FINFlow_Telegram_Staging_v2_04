# FINFlow v3 — Import Review Queue Architecture v1.24

## Назначение

Import Review Queue — защитный слой между старыми/сырыми источниками данных и живой системой FinFlow.

Старые расходы, доходы, смены, банковские операции, заметки из чатов и цели не должны попадать напрямую в Day Core, фонды, статистику или AI-советы. Сначала они становятся кандидатами на импорт.

## Почему это обязательно

Source 05 подтвердил риск: в проекте есть разные типы источников — чат-контексты, банковские PDF, текстовые логи, цели, старые расчёты, проектные решения. Часть данных является фактом, часть — идеей, частью — целевым значением, частью — устаревшим черновиком.

Если импортировать всё автоматически, FinFlow начнёт считать неверно: цели могут стать расходами, планы могут стать фактами, банковские переводы могут стать доходом, а личные данные могут попасть в публичный проект.

## Принцип

```
raw source → normalized candidate → review queue → approved entity → Day Core / Funds / Stats / AI
```

## Статусы кандидата

- `new` — найден новый кандидат.
- `needs_review` — требуется ручная проверка.
- `approved` — подтверждён пользователем или надёжным правилом.
- `rejected` — не импортировать.
- `merged` — объединён с другим кандидатом.
- `archived` — сохранён для истории, но не участвует в расчётах.

## Риски

- `low` — низкий риск, например каноническое обязательство из locked decisions.
- `medium` — похож на факт, но нужен контекст.
- `high` — высок шанс ошибки или дубля.
- `sensitive` — банковские операции, личные данные, токены, raw-чаты, private_raw_data.

## Жёсткое правило

Кандидат с риском `sensitive` или `high` не применяется автоматически.

## Что кандидат должен хранить

- источник;
- тип сущности;
- статус;
- риск;
- уверенность;
- дату;
- сумму;
- предложенную категорию;
- предложенный день;
- редактированный/обезличенный excerpt;
- причину проверки;
- возможные дубли;
- целевое действие: create/update/merge/ignore/manual decision.

## Минимальная реализация v1.24

В код добавлена модель:

```
src/lib/import-review/importReviewQueueModel.ts
```

В UI добавлен безопасный демонстрационный блок:

```
src/components/import-review/ImportReviewQueuePanel.tsx
```

Он не импортирует реальные банковские данные и не хранит секреты. Он показывает архитектуру очереди и принцип, что любые сырые данные сначала требуют review.
