## Важная честная фиксация

Это не финальная бухгалтерская таблица. Это **review layer**: машинно найденные кандидаты, которые нужно подтвердить через Import Center.

Причина: в исторических файлах одна и та же сумма может означать фактический расход, цель, пример расчёта, итог дня, остаток, перевод между своими счетами или комментарий. Поэтому FinFlow должен хранить `raw → candidate → review → confirmed`, а не без проверки превращать всё в боевые данные.

## Принцип старых данных

Старые доходы, расходы, смены, бензин, Drivee, фонды и банковская выписка НЕ теряются. Они становятся обучающим и импортным слоем FinFlow.

## Безопасность

`private_raw_data` и банковские/секретные источники нельзя загружать в GitHub, облако, Vercel, публичные архивы или Supabase Storage без отдельного решения.

Файлы в `docs/project/import/normalized` содержат редактированные фрагменты и кандидаты; всё равно перед публикацией проекта наружу их нужно проверить.
